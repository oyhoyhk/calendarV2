C 컴파일러 - cc / gcc

실행파일 생성과정

  • 전처리 단계 - 전처리기(cpp)
    • #include, #define, #ifdef 등 #으로 시작하는 문법 사항이 적절히 전처리된 C언어 소스파일이 만들어짐
  • 컴파일 단계 - 컴파일러(cc1)
    • C 언어 소스 파일은 컴파일 과정을 거쳐 어셈블리(Assembly) 소스 파일이 됨
  • 어셈블 단계 - 어셈블러(as)
    • 어셈블러 소스 파일은 어셈블 과정을 거쳐 목적(Object) 파일이 됨
  • 링크 단계 - 링커(Id)
    • 목적 코드(Object Code)는 라이브러리와 링크되어서 실행 가능(Executable)한 파일이 됨.

gcc 컴파일 옵션

옵션 설명
-o
  • 컴파일 결과로 생성되는 파일의 이름을 명시적으로 지정함.
  • -o 옵션 뒤에 파일명을 지정함.
-c
  • 목적(Object) 파일을 생성함.
-I
  • 헤더 파일의 디렉토리 위치를 명시적으로 지정함(i의 대문자)
-l
  • 컴파일에 사용되는 라이브러리를 명시적으로 지정함.
-L
  • 라이브러리의 디렉토리 위치를 명시적으로 지정함.
-D
  • 매크로를 지정함.

컴파일 유틸리티 - make

Makefile의 구조

  • 구조 : Target - Dependency List (Tab) Command List
  • 특징
    • MakeFile : make 유틸리티의 컴파일 수행 규칙을 나열한 참조 파일
    • Target : 생성할 결과
    • Dependency List : Target을 만드는데 관련이 있는 파일들의 목록
    • Command List : Target을 만들기 위해 사용될 명령어 목록

make 유틸리티 정리

  • 일정한 규칙을 지정하여 타겟파일을 만들어낼 수 있다.
  • 컴파일 명령어 처리를 쉽게 해주며 소스 모듈의 일부를 수정했을 때에도 전체적으로 일관성을 유지하도록 해준다.
